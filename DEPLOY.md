# Deployment Guide — Webflow + AI Pipeline Visualization

Two deployment paths are documented here. Choose the one that matches your situation:

| Path | Best for | Time |
|------|----------|------|
| [A — Embed into existing Webflow site](#path-a--embed-into-an-existing-webflow-site) | Publishing as a blog post on jkjrdev.com | ~30 min |
| [B — Webflow Cloud standalone app](#path-b--webflow-cloud-standalone-app) | A dedicated URL at its own path or subdomain | ~45 min (works on free tier) |

Both paths use the same three core files: `index.html`, `styles.css`, `script.js`. Path A also deploys `editorial.html` / `editorial.css` / `editorial.js` as a second page.

---

## Prerequisites

1. **A Webflow account** — a free Starter plan works for Path B; Path A requires a site on a paid plan if you want a custom domain
2. **Node.js 18+** installed locally (required for the Webflow CLI in Path B)
3. **A public GitHub repository** to host CSS/JS assets for CDN delivery (both paths)

---

## Path A — Embed into an existing Webflow site

This path publishes the visualization as a page inside your Webflow site. The HTML body content goes in a Code Embed. CSS and JS are too large for Webflow's character limits (10K/8K), so they are served from jsDelivr using your GitHub repo as the source.

### Step 1 — Push files to a public GitHub repo

```bash
git init
git add index.html styles.css script.js editorial.html editorial.css editorial.js
git commit -m "Add Webflow+AI pipeline visualization"
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git push -u origin main
```

### Step 2 — Get jsDelivr CDN URLs

jsDelivr serves any file from a public GitHub repo instantly with no setup. The URL pattern is:

```
https://cdn.jsdelivr.net/gh/USERNAME/REPO@BRANCH/FILENAME
```

Your four asset URLs (replace `YOUR_USERNAME` and `YOUR_REPO`):

```
https://cdn.jsdelivr.net/gh/YOUR_USERNAME/YOUR_REPO@main/styles.css
https://cdn.jsdelivr.net/gh/YOUR_USERNAME/YOUR_REPO@main/script.js
https://cdn.jsdelivr.net/gh/YOUR_USERNAME/YOUR_REPO@main/editorial.css
https://cdn.jsdelivr.net/gh/YOUR_USERNAME/YOUR_REPO@main/editorial.js
```

Verify each URL loads the correct file in your browser before continuing.

> **Tip:** Use a commit SHA instead of `@main` to pin to an exact version and prevent unintended changes from invalidating the CDN cache:
> `https://cdn.jsdelivr.net/gh/YOUR_USERNAME/YOUR_REPO@abc1234/styles.css`

### Step 3 — Create a new Webflow page for the interactive version

1. Open your Webflow Designer
2. In the Pages panel, click **+** → **New Page**
3. Name it (e.g., `webflow-ai-pipeline`) — this sets the URL slug to `/webflow-ai-pipeline`
4. Open **Page Settings** (gear icon in the Pages panel)

### Step 4 — Add CSS to the page head

In **Page Settings → Custom Code → Before `</head>` tag**, paste:

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/YOUR_USERNAME/YOUR_REPO@main/styles.css">
```

Click **Save**.

> **Note:** If you want the CSS to apply site-wide instead (e.g., to share tokens with other pages), add it to **Site Settings → Custom Code → Head**. For a single article page, page-level is cleaner.

### Step 5 — Add the body content as a Code Embed

1. In the Designer canvas, add an **HTML Embed** element that spans the full page width (remove default padding on the page body if needed)
2. Paste the entire contents of `index.html` between the opening `<body>` and closing `</body>` tags — everything inside `<article class="article">` and its surrounding elements, but **not** the `<html>`, `<head>`, or `<body>` tags themselves

The character limit per Code Embed is **8,192 characters**. The HTML body of `index.html` is minimal (the pipeline nodes and sections are generated by `script.js`), so it fits comfortably. If the embed editor shows a character-count warning, split the content across two stacked embed elements.

### Step 6 — Add JS to the page footer

In **Page Settings → Custom Code → Before `</body>` tag**, paste:

```html
<script src="https://cdn.jsdelivr.net/gh/YOUR_USERNAME/YOUR_REPO@main/script.js"></script>
```

Click **Save**.

### Step 7 — Publish and verify

1. Click **Publish** in the Webflow Designer
2. Open the live URL: `https://yourdomain.com/webflow-ai-pipeline`
3. Verify the pipeline renders, each stage node is clickable, and the detail panel opens
4. Test keyboard navigation (Tab to focus a node, arrow keys to move between stages)
5. Toggle your OS to dark mode and confirm the dark theme applies

### Step 8 — Repeat for the editorial version

Repeat Steps 3–7 for `editorial.html`:

- Create a second page (e.g., slug: `/webflow-ai-pipeline-article`)
- In **Page Settings → Head**, add `editorial.css` from jsDelivr **and** the Google Fonts link:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Playfair+Display:ital,wght@0,700;0,800;1,700&family=Inter:wght@400;500;600&display=swap" rel="stylesheet">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/YOUR_USERNAME/YOUR_REPO@main/editorial.css">
```

- Add the article HTML body content in a Code Embed
- Add `editorial.js` in the page footer
- Update any `href="index.html"` links inside the embed to point to `/webflow-ai-pipeline`

### Step 9 — Cross-link the two pages

Update the "View the interactive version →" link in the editorial embed to point to your live Webflow URL:

```html
<!-- Change this: -->
<a href="index.html" ...>View the interactive version →</a>

<!-- To this: -->
<a href="/webflow-ai-pipeline" ...>View the interactive version →</a>
```

## Path B — Webflow Cloud standalone app

This path deploys the files as a standalone static application on Webflow Cloud, served from Cloudflare's edge. It gives you a dedicated URL independent of your main Webflow site.

### Free tier compatibility

Steps 1–3 and Step 5 work on the **free Starter plan**. Step 4 (custom domain) requires a paid plan — see the callout on that step. Here's what you get on free:

| Resource | Free Starter limit |
|----------|-------------------|
| Cloud projects per site | 5 |
| Monthly visits (site-wide) | 1,000 |
| Deployment log retention | 1 hour |
| SQLite storage | 100 MB |
| KV reads/day | 100K |
| Object storage | 1 GB |
| Custom domain | **No** — `.webflow.io` subdomain only |

For a low-traffic portfolio piece or demo, the free tier is sufficient. The app mounts at a path on your `*.webflow.io` site (e.g., `yoursite.webflow.io/pipeline`).

### Step 1 — Install the Webflow CLI

```bash
npm install -g @webflow/webflow-cli
```

Verify:

```bash
webflow --version
```

### Step 2 — Log in to Webflow

```bash
webflow auth login
```

This opens a browser window for OAuth. Complete the login and return to the terminal.

### Step 3 — Wrap the files in a minimal static config

Webflow Cloud currently targets Next.js and Astro apps for full-stack deployment. For a plain HTML/CSS/JS project, the simplest wrapper is a minimal Next.js static export — or use a `webflow.json` config if your CLI version supports static-only mode.

**Option B1 — Static-only config (if supported by your CLI version):**

Create `webflow.json` in the project root:

```json
{
  "project": "webflow-pipeline",
  "type": "static",
  "publicDir": ".",
  "entryPoints": ["index.html", "editorial.html"]
}
```

Then deploy:

```bash
webflow cloud deploy -e production --auto-publish
```

**Option B2 — Minimal Next.js wrapper (most compatible):**

1. Scaffold a new Next.js app alongside the existing files:

```bash
npx create-next-app@latest pipeline-app --no-typescript --no-eslint --no-tailwind --no-app --no-src-dir
```

2. Copy your static files into `pipeline-app/public/`:

```bash
cp index.html editorial.html styles.css script.js editorial.css editorial.js pipeline-app/public/
```

3. In `pipeline-app/next.config.js`, set static export:

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
};

module.exports = nextConfig;
```

4. Build and verify locally:

```bash
cd pipeline-app
npm install
npm run build
npx serve out
```

Open `http://localhost:3000/index.html` and confirm the visualization loads.

5. Deploy to Webflow Cloud:

```bash
webflow cloud deploy -e production --auto-publish
```

### Step 4 — Set a custom domain (paid plan required)

> **Free tier:** Skip this step. Your app is already live at `yoursite.webflow.io/pipeline` after Step 3. The `.webflow.io` URL works for demos and portfolio links. Custom domains require at least the **Basic plan** ($14/mo).

1. In your Webflow dashboard, open the Cloud project settings
2. Under **Domains**, add `pipeline.yourdomain.com` (or your chosen subdomain)
3. Add the DNS CNAME record at your registrar:
   - **Name:** `pipeline`
   - **Value:** the Webflow Cloud edge URL shown in the dashboard
4. Wait for DNS propagation (typically 5–30 minutes)

### Step 5 — Verify the live deployment

**On a paid plan with a custom domain:**

1. Open `https://pipeline.yourdomain.com/index.html`
2. Open `https://pipeline.yourdomain.com/editorial.html`

**On the free Starter plan:**

1. Open `https://yoursite.webflow.io/pipeline/index.html`
2. Open `https://yoursite.webflow.io/pipeline/editorial.html`

**Both:**

3. Confirm both pages load fully and the inter-page links between them work
4. Run Chrome DevTools Lighthouse on both URLs — target 90+ Performance

## Post-deployment: Claude MCP audit

After either deployment path, run a site audit from Claude Desktop or Cursor (with the Webflow MCP configured):

```
Audit the site at https://yourdomain.com/webflow-ai-pipeline — 
check for missing alt text, broken links, and any console errors.
```

The **Site Audit**, **Link Checker**, and **Asset Audit** Claude Skills auto-activate and scan the deployed page without requiring explicit tool calls.

See [Getting started with Webflow MCP](https://developers.webflow.com/mcp/reference/getting-started) for OAuth setup.

## Character limit reference

| Location | Limit | Strategy if exceeded |
|----------|-------|----------------------|
| Site Settings → Head | 10,000 chars | Use `<link href="...">` from CDN instead |
| Site Settings → Footer | 10,000 chars | Use `<script src="...">` from CDN instead |
| Page Settings → Head | 10,000 chars | Same as above |
| Code Embed element | 8,192 chars | Split into two stacked embeds, or host HTML as a partial from CDN |

`styles.css` is ~15K characters and `script.js` is ~30K — both exceed Code Embed limits. Always load them via `<link>` and `<script src>` rather than pasting inline. The jsDelivr CDN URL approach in Path A handles this correctly.

## Troubleshooting

**Pipeline nodes don't appear**
The nodes are generated by `script.js`. Check that the JS loaded (Network tab in DevTools). If the `<script src>` URL returns a 404, the CDN URL or file path is wrong.

**Dark mode not working in Webflow**
Webflow's Designer preview overrides `prefers-color-scheme`. Test dark mode in the published site, not the preview canvas.

**Detail panel doesn't open on click**
The click handler delegates from `#pipeline`. If the embed uses a wrapper `<div>` with `pointer-events: none` or a Webflow interaction that intercepts clicks, remove the conflicting property.

**GSAP / Webflow Interactions conflicting with CSS animations**
If you later add GSAP or Webflow Interactions to the page, disable the `pulseDot` keyframe animation in `styles.css` (it uses `animation:` on `.connector__pulse`) to avoid conflicts with ScrollTrigger's tick loop.

**jsDelivr serving a cached old version**
jsDelivr caches by commit SHA. After pushing changes, either update the SHA in the `@SHA` CDN URL, or append `?v=2` to bust the cache temporarily.
