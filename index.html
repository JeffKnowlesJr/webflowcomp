Why HTML,
CSS & JS?

You start with a Webflow template. You spend hours customizing it — rewriting the CMS structure, building filters, fixing animations, extracting scattered custom code. The ecosystem has grown tools that eliminate chunks of this work. But some of it overlaps with what you already own (AWS, Cloudflare, Firebase).

Finsweet Attributes eliminates filtering boilerplate. GSAP (now free) eliminates animation work. MCP in Cursor eliminates content tedium. App Gen eliminates feature scaffolding. Code Components have constraints (Shadow DOM, 50MB, no server secrets) that matter when you have better options. The question isn't whether to use Webflow tools. It's which layers of your stack each tool actually improves.

💡
The decision framework: Use Finsweet/GSAP/MCP first. If you need a backend, use your existing infrastructure + Webflow's Data API. If you want an interactive feature, use App Gen. Code Components only when React needs to live inside the Designer for a client.
