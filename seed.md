**What it is:** A hosting platform for full-stack Next.js and Astro applications, deployed alongside your Webflow site on Cloudflare's edge network.

### Technical capabilities

- **Edge performance:** Sub-50ms global response times via Cloudflare Workers
- **Storage options:**

  - Key-Value Store — edge-cached, fast unstructured data

## 🔌 Domain C: Data API v2 — Programmatic Site Management

**Base URL:** `https://api.webflow.com/v2/`
**CDN URL:** `https://api-cdn.webflow.com/v2/` (read-only, cached)
**SDK:** `npm install webflow-api` (Node.js)

### CMS API — full CRUD

| Resource | Endpoints |
| :--- | :--- |
| Collections | List, Get, Create, Delete |
| Collection Fields | Create, Update, Delete |
| Staged Items | List, Get, Create (single + bulk), Update (single + bulk), Delete (single + bulk), Publish |
| Live Items | List, Get, Create, Update (single + bulk), Unpublish |

**Key concept — staged vs. live:** Items exist in two states. Staged = draft (not visible on site). Live = published. You can CRUD staged items without affecting the live site, then explicitly publish.

**Locale support:** Pass `cmsLocaleIds` parameter to create/retrieve items across multiple languages.

### Webhook events (real-time)

Subscribe to any of these on a collection:
- `collection_item_created`
- `collection_item_changed`
- `collection_item_deleted`
- `collection_item_published`
- `collection_item_unpublished`
- `memberships_user_account_added`
- `memberships_user_account_updated`

## ⚛️ Domain E: Code Components — Full Technical Reference

**What they are:** React components imported into Webflow and used like native elements — draggable, configurable via props, bindable to CMS fields.

### Capabilities

- Server-side rendering (SSR) — SEO and AEO ready
- CMS field binding — prop values can come from CMS collection fields
- Cross-site reuse — share one library across multiple Webflow projects
- Shared state between instances — via custom events, `localStorage`, or `sessionStorage`
- A/B test integration — add code component instances to Webflow Optimize tests
- Slots — nest native Webflow elements inside React components
- CSS variables — read Webflow design variables via `var(--token-name, fallback)`

### Critical limitations

| Limitation | Detail |
| :--- | :--- |
| React only | No Vue, Svelte, Angular |
| 50MB bundle cap | Entire library, not per-component |
| Shadow DOM | Site-level CSS classes don't apply. Use component-specific classes |
| No versioning | Each deploy replaces the entire library |
| No server-side secrets | All API calls are client-side only. CORS must allow cross-origin |
| No fragments | Components must have a single root element |
| No global state | React Context doesn't cross component boundaries. Use events/localStorage |
| Plan required | CMS, Business, or Enterprise site plan + Freelancer+ workspace |

### Key CLI workflow

```bash
npm i --save-dev @webflow/webflow-cli @webflow/react @webflow/data-types
npx webflow library share                  # bundle + deploy to Webflow
npx webflow library bundle --dev            # local test build
npx webflow library bundle --output-path ./build  # custom output
```

### State management patterns

Since React Context doesn't cross code component boundaries:
- Use **custom browser events** to communicate between component instances
- Use **`localStorage`/`sessionStorage`** for persistent cross-component state
- Use **URL parameters** for simple state (e.g., `?filter=blue`)
- Use external state libraries that work with isolated React roots

### Debugging in Shadow DOM

- Enable source maps for readable stack traces
- Use `--dev` flag for development builds with better error messages
- Inspect the Shadow DOM directly in Chrome DevTools
- `console.log` inside components works normally

## 🎬 Domain F: GSAP Integration — Native Animation Engine

**Big news:** Webflow **acquired GSAP** (GreenSock Animation Platform) in late 2024. As of **April 30, 2025, all GSAP features are 100% free** — including previously paid Club GreenSock plugins (SplitText, ScrollSmoother, ScrambleText, etc.).

### Two integration modes

**1. No-code: Webflow Interactions with GSAP**
- New version of Webflow Interactions, replatformed on GSAP under the hood
- Horizontal timeline UI (industry standard for motion designers)
- **ScrollTrigger** — scroll-driven animations, pinned sections, progress bars
- **Staggers** — sequential entrance animations for groups of elements
- **SplitText** — animate by character, word, or line
- Reusable interactions — define once, apply across site
- Class-based animations — add/remove/toggle CSS classes as animation actions
- Advanced targeting — CSS combinators (descendants, ancestors, siblings)
- Available on all plans, including Starter — no cost

**2. Custom code GSAP**
Enable in Site Settings → Integrations → GSAP. Then write in `<head>` or Embed components:

```html
<script>
  window.addEventListener("DOMContentLoaded", () => {
    gsap.from(".hero-title", { duration: 1, y: -50, opacity: 0 });
    ScrollTrigger.create({ ... });
  });
</script>
```

Plugins are auto-registered — no separate import needed after enabling in settings.

### Plugins now free (previously paid Club GreenSock)

- **SplitText** — text splitting for letter/word/line animations
- **ScrollSmoother** — smooth scrolling with lag effects
- **ScrollTo** — animate scroll position programmatically
- **ScrambleText** — randomized character scramble effect (coming to visual builder)
- **DrawSVG**, **MorphSVG**, **MotionPath** — advanced SVG animations

### When custom GSAP beats native Interactions

Use custom GSAP code for the 20% of cases needing:
- Precise scroll percentage scrubbing
- Vector morphing between SVG shapes
- Physics-based motion / spring effects
- Mouse velocity reactions
- Orchestrating 50+ elements with granular performance control
- Programmatically triggering visually-built interactions from code

Webflow + GSAP roadmap includes native Lottie and Rive integrations for After Effects and interactive animations.

## 🛠️ Domain I: Designer Extensions — Custom Tools Inside Webflow

**What they are:** Web apps (iframes) that run inside the Webflow Designer panel. Built with HTML/CSS/JS (or React/TypeScript via CLI templates). Control the canvas programmatically.

### Designer API objects you can control

| Object | What you can do |
| :--- | :--- |
| **Elements** | Create, insert, modify, delete DOM elements on canvas |
| **Styles** | Create/update/apply CSS classes |
| **Components** | Create/modify reusable component definitions |
| **Variables** | Define/update design tokens (colors, fonts, sizes) |
| **Pages** | Get/update page properties, SEO settings, site structure |

### How they work technically

- Client-side JavaScript APIs running in browser via iframe
- Interact with Webflow Designer exactly as a user would — in real time
- Reference existing objects with GET methods (`webflow.getSelectedElement()`)
- Create new objects with CREATE methods (`element.after()`)

### Build + deploy workflow

```bash
webflow extension init my-tool react        # scaffold React extension
webflow extension serve                     # dev server on port 1337
# Build and upload:
webflow extension bundle                    # creates bundle.zip (max 5MB)
# Upload via Workspace Settings → Integrations → App Development
# Submit to Webflow Marketplace for public distribution
```

### What you can build

- **Design system auditor** — scan site for inconsistent styles, flag non-token colors
- **Content populator** — fill pages/components with data from external spreadsheet/API
- **Layout automator** — generate page structures from a template with one click
- **Brand enforcer** — apply correct variables and classes site-wide from a sidebar panel
- **Asset manager** — bulk rename, tag, or replace images

## 🧪 Domain J: Webflow Optimize — A/B Testing & Personalization with Code

**What it is:** Native A/B testing and personalization platform, integrated into Webflow hosting infrastructure.

### Three modes of operation

| Mode | How it works |
| :--- | :--- |
| **Traditional A/B Test** | Fixed traffic split, single winner after statistical significance |
| **AI-Optimized Test** | Dynamic traffic allocation using ML, runs indefinitely, add variants post-launch |
| **Personalization** | Audience segments (firmographic, behavioral), 1:1 Enterprise personalization |

### Browser API (no installation — auto-enabled on all Webflow sites)

All methods available on the global `wf` object:

```javascript
// Always wrap in wf.ready()
wf.ready(function() {

  // Track which variation a visitor sees
  wf.onVariationRecorded(function(result) {
    console.log(result);  // send to your analytics
  });

  // Set custom attributes for targeting/personalization
  wf.setAttributes("user", {
    userSegment: 'enterprise',
    userRole: 'technicalBuyer',
    plan: 'pro'
  });

});
```

### Implementation options

1. **Via Webflow Custom Code** — paste `wf.ready()` block in `<head>` (before DOM renders, never missed)
2. **Via Google Tag Manager** — configure as a custom tag
3. **Multiple callbacks** — add multiple `wf.ready()` calls on the same page; execute in order

### Code integration points

- Call `wf.setAttributes()` to define visitor characteristics → feeds personalization logic
- Call `wf.onVariationRecorded()` to pipe variation data to analytics (GA4, Segment, etc.)
- Add code component instances as variants in A/B tests

## 💳 Domain K: Ecommerce + Stripe — Custom Checkout Code

### Three integration patterns

| Pattern | Complexity | Control | Use case |
| :--- | :--- | :--- | :--- |
| **Payment Links** | Low | Low | Simple one-time payments, max 20 line items |
| **Checkout Sessions API** | High | Full | Custom checkout UX, up to 100 line items, webhooks |
| **Third-party apps** | Medium | Medium | No-code Stripe via Flowout or similar |

### Checkout Sessions pattern (server-side required)

1. Server creates a Checkout Session via Stripe API
2. Client redirects to Stripe-hosted checkout (or custom via Stripe.js Elements)
3. Stripe calls your webhook on `checkout.session.completed`
4. Server fulfills the order

**Why client-side callbacks aren't enough:** Browser closes, async payments pending — webhook handler is the only reliable fulfillment signal.

### Webflow Ecommerce native customization

From the Designer, you can customize:
- Cart animations (easing, duration, item preview count)
- Cart states (default, empty, error messages)
- Direct checkout vs. cart-based checkout ("Buy now" button)
- Cart wrapper styling and overlay appearance

### Requirements

- Verified Stripe business account
- Stripe keys: publishable key (client), secret key (server), webhook signing secret
- Webflow paid Ecommerce plan (for native cart/checkout UI)
- Webflow Cloud (for server-side webhook handler and session creation)

## 🌐 Domain L: Headless Webflow — CMS as a Backend API

**Pattern:** Use Webflow purely as a content management system. The frontend is external (React, Next.js, Vue, Astro, whatever).

### Why you'd go headless

- Existing React/Next.js codebase you don't want to replace
- Complex application state that Webflow's native CMS pages can't handle
- Multi-channel content (web + mobile app + email) from one source of truth
- Custom rendering performance requirements

### Implementation approach

**Option 1 — Direct CMS API** (simplest)
External frontend calls `api.webflow.com/v2/collections/{id}/items/live` directly with a read-only site token.

**Option 2 — Custom API via Webflow Cloud** (recommended for production)
Build an Astro or Next.js API layer on Webflow Cloud that:
- Wraps the CMS API in your own endpoint schema
- Handles authentication (secrets stay server-side)
- Reshapes data (resolves option IDs, denormalizes references)
- Supports fuzzy search, pagination, custom filtering
- Adds caching for frequently read content

```javascript
// Astro endpoint: /src/pages/posts.json.js
export const config = { runtime: "edge" };

export async function GET({ request, locals }) {
  const KEY = locals.runtime.env.CMS_KEY;
  const res = await fetch(
    `https://api-cdn.webflow.com/v2/collections/${COLLECTION_ID}/items/live`,
    { headers: { Authorization: `Bearer ${KEY}` } }
  );
  const data = await res.json();
  return new Response(JSON.stringify(data.items.map(i => i.fieldData)), {
    headers: { 'Content-Type': 'application/json' }
  });
}
```

**CDN cache trick:** `api-cdn.webflow.com` caches identical requests for 5 minutes (2 min Enterprise). Cached calls don't count toward rate limits — critical for public APIs under load.

### Localization in headless mode

Pass `cmsLocaleIds` query param to retrieve items in specific locales. The Localization API also lets you update page DOM and metadata per locale via `POST /pages/{id}/dom`.

## 📝 Domain M: Custom Code Embeds — Limits, Strategies & Decision Tree

### Character limits (as of 2025)

| Location | Limit | Best for |
| :--- | :--- | :--- |
| Site Settings → Header | 10,000 chars | Global tracking scripts, cookie banners |
| Site Settings → Footer | 10,000 chars | Global deferred scripts, chat widgets |
| Page Settings → Header/Footer | 5,000 chars per field | Page-specific experiments, verification tags |
| Code Embed element | 8,192 chars (8KB) | Inline per-element custom HTML/CSS/JS |

**Important:** Character counts include spaces and minification doesn't help as much as you'd think with spaces.

### What you can and can't use

✅ HTML, CSS, JavaScript
✅ `<script src="...">` tags to external files (no character limit for external URLs)
✅ Webflow includes jQuery 3.5.1 by default — use `$()` without importing it
❌ No server-side languages (PHP, Python, Ruby)
❌ No `<html>`, `<head>`, or `<body>` wrapper tags (breaks site layout)
❌ Don't import a second jQuery — it conflicts

### Workarounds when you hit the limit

**1. Host code externally (best practice)**
Upload your JS/CSS to a CDN and load with a short tag:
```html
<script src="https://cdn.example.com/my-script.min.js"></script>
```
Options: jsDelivr (from GitHub), Cloudflare Pages, Vercel, your own Webflow Cloud endpoint.

**2. Multiple embed elements**
Spread code across multiple Code Embed elements on the page — each gets its own 8,192-char budget.

**3. Minify aggressively**
Strip comments, whitespace, newlines. Tools: Minifier.org, esbuild, Terser.

### Embed vs. Code Component decision tree

| Use custom code embed when... | Use code component when... |
| :--- | :--- |
| Simple script, one-off use | Reused across multiple pages/sites |
| Third-party library load (GTM, Intercom) | Complex React UI with props |
| Quick prototype / experiment | Needs CMS field binding |
| No React dependency | Server-side rendering needed |
| Character budget fits | Advanced state management needed |
| No TypeScript needed | Needs to be used by designers without code |
