Building a Website
with Webflow + AI

A field guide to the full Webflow ecosystem — 13 domains, 6 stages, and a decision framework for choosing the right approach every time.

View the interactive version →

The Webflow ecosystem has quietly grown into something most developers haven't caught up with. As of early 2026, it spans 13 distinct technical domains — from App Gen, which generates full-stack interactive applications from a natural language prompt, to Webflow Cloud, which hosts Next.js and Astro apps on Cloudflare's edge with sub-50ms response times. The question of how to build a website has been replaced by a decision tree: which layer of the stack does Webflow own, which does custom code own, and what does AI generate automatically?

This article maps that decision through six stages, from initial research to deployment. Each stage corresponds to a distinct phase of thinking and a distinct set of tools. The goal isn't to cover every feature — it's to help you pick the right tool for the right job, and avoid the expensive mistake of reaching for complexity before checking whether a simpler layer would have solved the problem in fifteen minutes.

01 Research

Map the landscape before you build anything

Research now means something different than it did two years ago. You're not just deciding whether to use Webflow — you're mapping a 13-domain ecosystem to determine which combination of tools fits your specific build. App Gen handles prompt-built interactive apps. Finsweet Attributes handles CMS filtering, pagination, and search with zero JavaScript. Wized connects Webflow to any REST API. Code Components embed React. Webflow Cloud hosts full-stack Next.js and Astro. Choosing the wrong tool at this stage — reaching for Code Components when Finsweet Attributes would do the job, or wiring up a custom API layer when App Gen would generate it — is the most expensive mistake you can make.

Two things have become essential pre-build reading. First: the Webflow changelog. New features ship constantly, and they frequently eliminate the need for custom code that was standard practice six months earlier. Second: the deprecation list. Webflow Logic was disabled June 27, 2025. Webflow Memberships is sunsetting. Any tutorial, StackOverflow answer, or agency documentation referencing either of these features is now misleading. Research means knowing what's current.

02 AI Generate

From a prompt to a production-ready foundation

App Gen changes the math on what's worth building manually. Launched at Webflow Conf in November 2025 and currently in free public beta on all plans, App Gen takes a natural language description and generates production-grade code grounded in your existing design system — your color variables, CMS schema, component library. The output is actual code, deployed to Webflow Cloud, connected to your CMS in real time. Supported app types include location finders, job boards, pricing calculators, event calendars, multi-step forms, business tools, and interactive games.

What separates App Gen from Bolt, Lovable, or v0 is the starting point. Those tools generate generic UI from a blank canvas. App Gen starts from your brand's operating system. Consistency comes built-in because it reads your design tokens before generating anything. Enable it via Workspace Settings → Webflow AI toggle. Enterprise sites can also toggle it per-site in Site Settings.

The Webflow AI Site Builder is a separate, older tool — good for scaffolding a multi-page site structure and a basic design system from a prompt (up to 5 pages). Use it for site scaffolding, App Gen for interactive functionality within those pages. Claude's MCP connector layers on 19 API tools and 9 Claude Skills for ongoing management — bulk CMS updates, site audits, link checking, safe-publish workflows. The Skills don't need explicit invocation; they auto-activate when they detect the right context.

AI tools at this stage

  • App Gen (public beta, free) — full-stack apps from a prompt, grounded in your design system and CMS
  • Webflow AI Site Builder — site scaffold from a prompt; up to 5 pages, no code needed
  • Claude MCP — 19 tools — Data and Designer API tools via OAuth; runs in Claude Desktop or Cursor
  • Claude Skills — 9 workflows — Bulk CMS Update, Site Audit, Asset Audit, Link Checker, Safe Publish, and more

Claude Skills auto-activate — you don't call them explicitly. They detect when they're relevant and surface themselves into the conversation.

03 Design

Design systems, animations, and collaboration — all free

Webflow acquired GSAP in 2024. The practical impact landed April 30, 2025, when every GSAP plugin became free — SplitText, ScrollSmoother, ScrambleText, DrawSVG, MorphSVG, all previously requiring a paid Club GreenSock subscription. The new Webflow Interactions are replatformed on GSAP under the hood, with a horizontal timeline UI that handles 80% of animation needs without code: scroll-driven animations via ScrollTrigger, staggered entrance animations for groups of elements, text animation by character, word, or line via SplitText. For the remaining 20% — physics-based motion, SVG morphing, precise scroll percentage scrubbing — write custom GSAP code in a script embed after enabling GSAP in Site Settings.

Design system setup is the foundational decision of this stage. Set your color, font, and spacing variables before designing any pages — they propagate everywhere. The most common mistake is designing pages before establishing variables, then spending hours manually correcting inconsistent values across the site. Real-time collaboration is live on all plans (up to 25 concurrent users), so design and development can run in parallel. Flexible components with conditional visibility — showing or hiding elements based on props, CMS fields, or locale — cut the number of component variants you need to maintain dramatically.

Design tools at this stage

  • GSAP — all plugins free — ScrollTrigger, SplitText, ScrollSmoother, MorphSVG, DrawSVG
  • GSAP + Webflow Resource Hub — full documentation, tutorials, and Webflow-specific guides
  • SVG Path Editor — Yqnn — create custom icons visually, then paste path data into Webflow
  • Flexible Components — conditional visibility driven by props, CMS fields, or locale settings

Set design token variables before designing any pages. Every page, component, and interaction reads from them — change one, change everything consistently.

04 Build

Four tool tiers, each with a different ceiling

The build stage spans four distinct tool tiers. The decision of which to use sets both the complexity and the ceiling of what you can ship. The rule of thumb: start at the bottom of the stack and only move up when the tier below genuinely can't do the job.

Finsweet Attributes is the first reach. 200 million loads per month. 30+ solutions. Zero JavaScript writing required — add HTML data- attributes to elements and the library handles everything: CMS filtering with AND/OR/grouped logic, load-more pagination, infinite scroll, fuzzy search, custom sort. If your requirement is CMS-driven interactivity, check Finsweet first. The entire library is open source and loaded via a single CDN script tag.

Code Components are next. React components that live in Webflow's Components panel, draggable onto any page, configurable via props, bindable to CMS fields, server-side rendered for SEO. They run in the Shadow DOM — site-level CSS classes don't apply inside them. This trips up most developers on their first build. State doesn't cross component boundaries (React Context only works within a single root), so inter-component communication happens via custom browser events or localStorage. Bundle cap is 50MB across the entire library, React only, no fragments.

Wized sits between the tiers. It's a JavaScript framework that wraps Webflow's output and adds a visual event/request builder for connecting to any REST API — Xano, Supabase, Airtable, Shopify GraphQL, Firebase. Teams have used it to build enterprise LMS platforms (60,000+ students, 1.2M+ records), AI-powered SaaS products, and headless Shopify stores without abandoning Webflow's visual editor. DevLink is for teams with an existing React codebase: webflow devlink sync exports Webflow components as production-ready React, and re-syncing propagates design updates to code without developer involvement.

Build tools at this stage

  • Finsweet Attributes v2 — 200M+ loads/month; 30+ solutions via HTML data-attributes, zero JS writing
  • Wized — app layer for Webflow; connects to any REST API without a backend rewrite
  • Code Components (React) — SSR, CMS-bindable, Shadow DOM; 50MB bundle cap, React only
  • DevLink — export Webflow designs as production React components; design-to-code sync
  • Data API v2 — full CRUD for CMS, pages, ecommerce, users; 7 webhook event types

Code Components run in the Shadow DOM. Site-level CSS classes don't apply inside them — a fact that costs most developers several hours to discover on their first build.

05 Enhance

Layer on what doesn't break the baseline

Enhancement is additive. Build the content first in a state that's readable and functional without JavaScript. Then layer in what doesn't break that baseline: A/B tests, personalization, on-device AI, richer animations. The principle sounds obvious but in practice it's where most progressive enhancement breaks down — features get built that depend on JavaScript for core functionality, then break for users who see a flash of unenhanced content on slow connections.

Webflow Optimize is a native A/B testing and personalization platform. The browser API is auto-enabled on all Webflow sites — no installation required. The critical placement detail: put wf.ready(callback) in the site-wide <head> custom code field, not in a page-level script or a Code Embed at the bottom of the page. It fires before the DOM renders, which means variation logic executes before any element is painted. Call wf.setAttributes("user", { plan: "pro", role: "developer" }) to define visitor segments for personalization targeting. Pipe variation data to your analytics with wf.onVariationRecorded().

Custom code embed limits are firm: 10,000 characters in the site-wide header or footer, 8,192 characters per Code Embed element. For larger scripts, host on jsDelivr (from a public GitHub repo), Cloudflare Pages, or a Webflow Cloud endpoint — the <script src> URL takes zero character budget. Two deprecations that redirect significant investment: Webflow Logic was disabled June 27, 2025 — replace with Webhooks + Make/Zapier, or serverless functions on Webflow Cloud. Webflow Memberships is sunsetting — replace with Memberstack for tiered membership and content gating.

Enhancement tools at this stage

  • Webflow Optimize — A/B testing and personalization; wf.ready() and wf.setAttributes() API
  • Chrome On-Device AI — enhance CMS content with no external API calls, no latency, no privacy trade-off
  • Conditional Visibility — show/hide by props, CMS fields, or locale; zero custom code required
  • Memberstack — replacement for deprecated Webflow Memberships; tiered content gating

Webflow Logic was disabled June 27, 2025. Any tutorial or agency playbook referencing it is now actively misleading.

06 Deploy

Audit before you publish, then ship to the edge

Webflow Cloud is not just hosting — it's a full-stack platform for Next.js and Astro applications deployed on Cloudflare's edge network. Sub-50ms global response times. Native storage: SQLite for relational data, Key-Value Store for edge-cached unstructured data, Object Storage for files. GitHub integration provides native CI/CD — commit triggers auto-deploy. The CLI pipeline: webflow library share --no-input deploys code components, webflow cloud deploy -e production --auto-publish deploys the app and publishes the Webflow site in a single command.

Before any publish, run the Claude MCP audit suite. The Site Audit, Asset Audit, and Link Checker Skills auto-activate and scan the entire site for missing alt text, broken links, inconsistent spacing, and underperforming metadata — the kind of sweep that would take a human several hours to do manually. The Safe Publish Skill shows a diff of what changed and runs pre-publish checks before committing. One CDN trick worth memorizing: for live CMS content reads, use api-cdn.webflow.com instead of api.webflow.com. The CDN URL caches identical requests for 5 minutes (2 minutes on Enterprise), and cached calls don't count against rate limits. For a public API built on top of Webflow CMS, this is the difference between hitting limits under load and serving traffic at scale.

For Stripe checkout: the Checkout Sessions pattern requires a server-side handler — Webflow Cloud is the natural home. The webhook on checkout.session.completed is the only reliable fulfillment signal. Browser-side callbacks fail when the browser closes before an async payment resolves. Build the webhook handler before the client-side UI; it's the load-bearing piece.

Deploy tools at this stage

  • Webflow Cloud — edge hosting for Next.js/Astro; SQLite, KV Store, Object Storage, GitHub CI/CD
  • Webflow CLI — webflow library share and webflow cloud deploy; fully CI/CD ready
  • Claude MCP — Audit Suite — Site Audit, Asset Audit, Link Checker, Safe Publish Skills
  • Webflow MCP — Getting Started — OAuth setup, tool inventory, Claude Skills reference

Why this page is built with vanilla HTML, CSS, and JavaScript

The interactive version of this article uses HTML, CSS, and JavaScript with zero dependencies, no build step, and no framework. This version adds one: a Google Fonts stylesheet for editorial typography. Neither uses React, a bundler, or a CSS preprocessor.

That's a deliberate argument. A polished, accessible, interactive experience doesn't require a framework. It requires understanding the platform. IntersectionObserver handles scroll tracking. CSS custom properties handle theming. CSS Grid and Flexbox handle layout. The Webflow ecosystem — App Gen, Code Components, Finsweet, Wized, Webflow Cloud — is the layer you reach for when native web tech genuinely isn't enough. Not before.

The decision framework applies to web development generally, not just to Webflow. Start with the simplest tool that does the job. Move up the complexity ladder only when you hit an actual ceiling. The ceiling is higher than most developers assume it is.
