# What the New Webflow Stack Actually Does for a Full-Stack Developer

You build Webflow sites from templates and spend hours customizing. You already use AWS, GCloud, Firebase, and Cloudflare. The question isn't whether Webflow is worth your time — it's **which of its 13 domains actually reduce manual work** and which overlap with what you already own.

**Below we show, not just tell.** Use the [interactive pipeline](index.html) and try the demos:

- **Filter a list** — Finsweet does this with data-attributes and zero custom JS. Click **Build** and type in the filter box.
- **Design tokens** — Change one variable, everything using it updates. Click **Design** and change the token dropdown.
- **Scroll-triggered reveal** — Same idea as GSAP ScrollTrigger. In **Design**, scroll the "Scroll me into view" block into view.
- **Shadow DOM** — Code Components run inside it; site CSS doesn't apply. In **Build**, see the Shadow DOM demo.
- **MCP in Cursor** — Sample prompt and output (we can't run it here). Click **AI Generate**.
- **API CDN** — Use `api-cdn.webflow.com` for CMS reads; cached, no rate limit. Click **Deploy**.

In the **Which approach?** table, click a row to expand a one-liner or **See demo** to jump to that stage. Use the demos to decide which tools are worth adding to your stack.
